# Architecture Decision Records

Each file documents a significant technical decision: what was chosen, what was
considered, and why.

## Conventions

- **Edit in place** when a decision changes. Git history is the audit trail.
- **Update the Status field** if a decision is fully retired or reversed:
  `Accepted` → `Superseded`.
- **Add a new ADR** only for decisions with real tradeoffs — not every library
  choice, only the ones a new contributor would reasonably question.

## Status values

- `Proposed` — under discussion
- `Accepted` — current decision
- `Superseded` — no longer applies; kept for historical context
