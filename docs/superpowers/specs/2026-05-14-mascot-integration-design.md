# Mascot Integration Design

**Date:** 2026-05-14
**Status:** Approved

## Overview

Integrate the CarroQueSí shopping-cart mascot (a 3D rendered cart with glasses giving a thumbs-up, transparent background PNG) across five surfaces: app icons, sign-in screen, list empty state, dashboard empty state, and invite screen.

## Asset

- **Source file:** `ChatGPT Image May 13, 2026, 07_02_55 PM.png` (user-supplied, transparent background)
- **Committed as:** `frontend/src/assets/mascot.png`
- **Format:** PNG with transparency — works on any background colour without modification

## Shared Component

Create `frontend/src/components/Mascot.tsx`:

```tsx
import mascotUrl from '../assets/mascot.png'

interface Props {
  size?: number
}

export function Mascot({ size = 160 }: Props) {
  return (
    <img
      src={mascotUrl}
      alt="Mascota de CarroQueSí"
      width={size}
      height={size}
      style={{ objectFit: 'contain' }}
    />
  )
}
```

No CSS file needed — sizing is fully controlled by the `size` prop.

## Surface-by-Surface Design

### 1. Sign-in Screen (`SignInScreen.tsx`)

Place `<Mascot size={160} />` above the `<h1>CarroQueSí</h1>`. The screen is already a centred flex column, so no layout changes are needed — just insert the component.

```
[Mascot 160px]
CarroQueSí          ← h1
Lista de compras…   ← subtitle
[Continuar con Google]
```

### 2. List Empty State (`ItemList.tsx`)

The existing empty state (lines 67–73) renders `"Sin productos — añade el primero abajo"`. Replace it with:

```
[Mascot 120px]
Sin productos todavía      ← short bold line
Añade el primero desde abajo  ← muted subtitle
```

Split the single `<p>` into two lines for visual hierarchy. Add a small `gap` to the centred flex container.

### 3. Dashboard Empty State (`DashboardScreen.tsx` + `CreateListCard.tsx`)

`DashboardScreen` passes `isFirst={lists.length === 0}` to `<CreateListCard>`. When `isFirst` is true, render a mascot above the create button inside `CreateListCard`:

```
[Mascot 120px]
Aún no tienes listas    ← new text, only when isFirst
[+ Crea tu primera lista]
```

Add the mascot and heading inside the `!expanded` branch of `CreateListCard`, guarded by `isFirst`.

### 4. Invite Screen (`InviteScreen.tsx`)

Add `<Mascot size={100} />` above the `.invite-screen__card` div in the `preview` state. The mascot sits outside and above the card, giving the screen an illustrated feel before the user sees the list details.

```
[Mascot 100px]
┌──────────────────┐
│  🛒  Compra…     │
│  Invitado por…   │
│  [Unirse]        │
└──────────────────┘
```

The list emoji inside the card is kept — it identifies the specific list, while the mascot above is the app brand.

### 5. App Icons

Generate new icons from `frontend/src/assets/mascot.png` using a Node script (`scripts/generate-icons.mjs`) with the `sharp` package.

| File | Size | Background |
|------|------|------------|
| `public/pwa-64x64.png` | 64×64 | transparent |
| `public/pwa-192x192.png` | 192×192 | white `#ffffff` |
| `public/pwa-512x512.png` | 512×512 | white `#ffffff` |
| `public/apple-touch-icon-180x180.png` | 180×180 | white `#ffffff` |
| `public/maskable-icon-512x512.png` | 512×512 | purple `#aa3bff`, mascot padded to 80% safe zone |
| `public/favicon.ico` | 32×32 | transparent (via `sharp` + `to-ico`) |

The script lives at `frontend/scripts/generate-icons.mjs` and is run from the `frontend/` directory. Add `sharp` as a dev dependency in `frontend/package.json`. The script is a one-off generator; running it overwrites the existing placeholder icons.

`favicon.svg` and `icon.svg` / `icons.svg` are left as-is for now — they serve as SVG fallbacks and are not visible in the installed PWA.

## What is Not Changing

- No new routes or API endpoints
- No changes to backend
- No changes to existing CSS variables or theme system
- The mascot does not appear on the loading/spinner states (only empty/first-run states)
- The mascot does not appear on error states (those remain text + retry button)
- Purchased item rules, filter bar, price sheets — untouched
